FLITE - Track Changes Plugin for CKEditor

Demo Page Web Site

Main Features

  • Enable / disable track changes mode
  • Show / hide tracked changes
  • Accept/Reject all/some changes
  • Accept/Reject a single change
  • Display customizable tooltips for each change

Getting Started

  • IMPORTANT: FLITE requires jQuery
  • Add the flite plugin to ckeditor. The simplest way to do this is by adding the following line to ckeditor's configuration (e.g. config.js, which is the default location of the configuration data). 
      config.extraPlugins = 'flite';

  • Optionally include flite-interface.js in your source, so you can use the various constants defined in it rather than string literals.

    <script type="text/javascript" src="ckeditor/plugins/flite/flite-interface.js"></script>

  • See the documentation for all the configuration options.

Configuration

The FLITE plugin is automatically activated after you install as described above. For the full details of tweaking the loading process, toolbar commands, users and more, see the documentation.

API

The FLITE plugin exposes a wide range of methods, events and properties for controlling and reporting the tracked changes. see the documentation for details.

Notes

  • The FLITE tracking engine adds markup to the editor content. All the change tracking data is stored inline, as part of the document.
  • FLITE tracks a wide range of editing actions, but not all of them. Most notably, style edits and search/replace operations are not tracked (they're on our roadmap, though).

Browser Compatibility

FLITE has been tested on Firefox 15+ and Chrome 13+. Support for MSIE 9+ is not guaranteed, although the current version seems to work on it.
