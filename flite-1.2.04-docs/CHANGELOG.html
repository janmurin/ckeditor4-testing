1.2.0 Mar 10 2019

Version 1.2.0 brings TinyMCE 5 compatibility, new and better tooltips (along with simpler tooltip options), better IME support and several bug fixes around deletion and caret placement. Be sure to read the documentation regarding FLITE.configuration, as some configuration fields have changed.

  • Added a new configuration field, styleUrls which allows you to inject the content of any css file into the edited document. The code currently translates the deprecated cssPath field into the new option, but this flow is likely to be removed in a few versions.
  • Added a new configuration field, allowQuitWithChanges. If set to true, FLITE does not require the user to resolve all changes before pausing tracking.
  • FLITE for TinyMCE now supports the commands configuration option
  • Resolved more quirks in IME support (complex input, e.g. Japanese)
  • Fixed issues with deletion of line breaks and paragraphs.
